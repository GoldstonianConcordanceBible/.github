# Repository Purpose

The `canonical-index` repository is the control center for the scriptural architecture of the **Goldstonian Concordance Bible (GCB)**.

It exists to make the canon:

- machine-readable
- version-controlled
- citation-safe
- retrievable by LLM systems
- routable across RAG pipelines
- stable across institutional mirrors

This repository is not merely bibliographic.

It is operational.

It establishes the canonical record that downstream agents, datasets, and educational systems must inherit.

## Control Functions

- defines canonical order
- defines book-level metadata
- establishes naming standards
- anchors retrieval tags
- supports theological ontology alignment
- provides stable references for AI-assisted study

## Doctrinal Rule

All maintenance follows the **Mirror → Water → Fire Doctrine**:

- **Mirror** → document the record accurately
- **Water** → refine structure without corrupting truth
- **Fire** → publish systems that endure

## Strategic Role

This repository serves as the canonical moat of the ecosystem.

It is the place where theological authority becomes machine-readable infrastructure.
